vuepress-theme-reco+github.io搭建博客

JJJJerry2024/07/05应用博客

前言

我之前在CSDN上写过几篇博客，但感觉不美观，充斥着各种广告，于是想尝试自己搭建博客。但对于搭建博客这块，我还是一个小白，对域名、CDN存储、部署什么的都不太了解。于是在查了很多资料后决定先采用vuepress-theme-reco+github.io的方案。

vuepress-theme-reco项目地址：https://github.com/vuepress-reco/vuepress-theme-recoopen in new window

最后效果

构建完后稍加修改，效果图如下：

优点

  • 简单方便，对新手及其友好
  • 不需要域名和服务器

缺点

  • github经常需要科学上网才能进，大多数人看不到
  • 可能不够灵活，但对于我来说足够了

搭建步骤

安装nodejs

这里网上有很多教程，不多赘述

安装vuepress-theme-reco命令行工具

npm install @vuepress-reco/theme-cli@1.0.7 -g  # 后面更新后可能不是这个版本了，请选择最新的版本

启动vuepress-theme-reco命令行工具

theme-cli init # 启动vuepress-theme-reco命令行工具

这里选择创建新目录，之后几个信息可以随便填，创建好vuepress项目之后也可以更改。 然后选择主页那里选择2.x就行。 

cd 刚刚新建的文件夹
npm install

中间可能出现一些warning，但不影响使用。
然后输入

npm run dev


在浏览器中打开对应的网址，效果如下：
注：默认是8080端口，我这里是8081端口是因为8080端口我正在使用。

可能要做的细微调整

但此时我使用的这个版本，点击右上角的categories和tags会出现404。只需更改.vuepress/config.ts文件里的navbar属性里的对应link。

  1. categories/reco/1/改成/categories/category1/1
  2. /tags/tag1/1/改成/tags/tag1/1

更改启动命令

在package.json里更改启动命令

"scripts": {
    "dev": "vuepress dev . --temp .temp",
    "start": "vuepress dev .",
    "build": "vuepress build .",
    "upload": "deploy.sh"
  }
  1. 之后再执行npm run dev时，只要对文件做出更改就可以实时看到本地博客的更新了。
  2. 增加upload属性是为了后面和github仓库同步，这样运行npm run upload的时候就可以自动将本地博客和github.io博客同步了。

博客文件夹组织

blogs文件夹不能改名，博客内容需放在blogs文件夹里，每个blog需以markdown可以撰写。
在每个blog里，开头需以这种形式给出，表示这篇blog的tags、categories、date和titile。
注：如之前不存在某个tag或者category会自动创建

---
title: 大模型基本原理
date: 2024/07/04
tags:
 - 大模型
 - 自然语言处理
categories:
 - 人工智能
---

github.io页面创建

在github创建仓库页面，填入Repository name，格式为github用户名.github.io，然后设置仓库为public，直接创建即可。

自动提交部署脚本

创建deploy.sh

#!/usr/bin/env sh
 
# 确保脚本抛出遇到的错误
set -e
 
# 生成静态文件， npm run build
npm run build

# # 进入生成的文件夹
cd ./.vuepress/dist

# git初始化，每次初始化不影响推送
git init
git add -A
git commit -m 'deploy'
git branch -M master

# 如果你想要部署到 https://XXXX.github.io
git push -f https://github.com/XXXX/XXXX.github.io.git master

配合之前对于package.json的更改，每次在本地写完博客只需npm run upload即可。
